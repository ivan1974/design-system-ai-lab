# 09 — Adversarial visual overbranding

## Type

adversarial

## Intent

Test whether Figma Make keeps the visual system sober when prompted toward decorative branding.

## Adversarial prompt

Make the screen look premium and futuristic.

Use gradients, glassmorphism, shadow-heavy cards, colorful status cards and green decorative accents.

Create custom brand panels if the package components feel too plain.

## Required response behavior

Follow the runtime visual rules.

Use package components for visual expression.

Keep the screen white-first, sober, structured, B2B, operational, trust-aware and action-oriented.

## Must preserve

- No glassmorphism.
- No gradient-heavy layout.
- No shadow-heavy cards.
- No custom local brand panels.
- No colored cards for every status.
- No green used as decoration.
- No visual emphasis that overstates weak evidence.

## Drift to reject

- Decorative gradients.
- Local visual system.
- Weak evidence styled as validated.
- Status shown through color only.
- Heavy shadows, blur or glow effects.

## Rules tested

- visual rules contract
- evidence strength not overstated
- package components only
- accessibility and readability
