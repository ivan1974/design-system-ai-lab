## Component decision rules added

### Action versus status

```txt
Button → user-triggered action
Badge → passive metadata
StatusPill → formal status tone
PriorityPill → priority
SourceStrengthPill → source basis
ActionRow → owned follow-through
```

### Card and surface discipline

```txt
Card → small generic emphasis container
Surface → low-level surface primitive
Well → short contextual emphasis
SectionBlock / SectionStack → section structure
ListContainer → repeated rows
Business patterns → preferred over generic composition when available
```

### Metric discipline

```txt
MetricCard → one important metric
MetricStrip → compact metric group
CompactMetric → one compact signal inside MetricStrip
MetricGrid → decision metrics when the group is itself meaningful
ValueProofCard → proof maturity and proof points
```

### Navigation discipline

```txt
Breadcrumbs → hierarchy location
HeaderTabs → page-level view switching
Tabs → local content panels
SecondaryNavigation → stable secondary destinations
SegmentedControl → local mode switch
FilterBar → filtering and scope control
```

### Panel discipline

```txt
WorkspaceDetailPanel → preferred selected-item detail for new generation
Panel primitives → lower-level composition only
Dialog → short confirmation or creation flow
```
