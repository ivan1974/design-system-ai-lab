# v0.5.1 Sprint 11 — Review and Repair Alignment

## Purpose

Sprint 11 aligns review guidance and repair prompts into one operational loop for generated screens.

A reviewer should be able to move from issue detection to the right repair prompt without guessing.

## Review loop

```txt
1. Run the required review checklist.
2. Route failed checks through repair-router.md.
3. Apply the matching repair prompt.
4. Re-run review.
5. Continue with workspace and quality review.
6. Compare with the closest golden example.
```

---

## Severity model

```txt
blocking = must repair before first-draft acceptance
quality = valid draft, improve when useful
acceptance = final confirmation before handoff
```

---

## Main changes

```txt
Review docs now use v0.5.1 vocabulary.
Every required review area has a repair route.
The repair router maps review issues to actual repair prompt files.
Controlled values include customerReadiness and humanValidation.
Forms and accessibility are included in the review loop.
Golden example comparison includes the installed-base example.
```
