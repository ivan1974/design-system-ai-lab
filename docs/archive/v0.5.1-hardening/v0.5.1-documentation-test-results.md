## New coverage

The new test verifies:

```txt
props contract components are exported
approved imports are exported or explicitly legacy-rationalized
preferred components have guidelines
preferred component guidelines use required GenAI sections
controlled values in component guidelines match props.contract.json
setup.md mirrors the global prop value rule
component references in guidelines resolve to known package components or allowed exported types
prompts do not reference obsolete component choices
```

---

## Drift now covered

Sprint 12 makes these drifts test-detectable:

```txt
contract component missing from public exports
approved import missing from public exports
preferred component without guideline
preferred guideline missing required GenAI sections
controlled value absent from a documented guideline
setup.md missing a global controlled prop family
guideline referencing a non-existent package component
prompt referencing removed or obsolete component choices
```

---

## Rationale

The test intentionally distinguishes:

```txt
preferred components
legacy or use-with-care components
shared guideline coverage
public exported types
```

This avoids failing when a component remains exported for compatibility but is no longer preferred for new generation.
