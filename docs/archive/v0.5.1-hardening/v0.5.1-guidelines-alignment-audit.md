# v0.5.1 Guidelines Alignment Audit

## Purpose

This audit is the starting point for `v0.5.1 — GenAI Documentation Hardening`.

Goal:

```txt
No exported component should be invisible to documentation, stories, contracts or GenAI guidance.
```

This file is intentionally strict.

A component is not considered ready for GenAI unless the system can answer these questions without guessing:

- Is the component exported?
- Is it present in Storybook?
- Is it documented in guidelines?
- Is it listed in the component contract?
- Are its important props documented?
- Are controlled values documented?
- Are GenAI generation rules documented?
- Are anti-patterns documented?

---

## Audit sources

Primary sources:

- `src/design-system/index.ts`
- `contracts/components.contract.json`
- `contracts/props.contract.json`
- `guidelines/components/`
- `guidelines/forms/`
- `guidelines/decision/`
- `guidelines/patterns/`
- `guidelines/screen-architecture/`
- `guidelines/review/anti-patterns.md`
- `.storybook/main.ts`

Storybook is configured to load:

```txt
../src/**/*.mdx
../src/**/*.stories.@(js|jsx|mjs|ts|tsx)
```

Sprint 0 does not mark story coverage as ready unless it is explicitly verified.

Default story status in this audit is therefore:

```txt
Not confirmed — treat as a gap until Sprint 2 proves coverage.
```

---

## Status scale

Use these statuses:

```txt
OK
Needs update
Missing guideline
Missing story proof
Contract mismatch
Props gap
Deprecated / use with care
Remove or consolidate
```

A component can have several issues.

---

## Critical findings

### P0 — Contract mismatches

These must be resolved first because they create ambiguity for GenAI and tests.

- `CompactMetric` is exported and used by guidance, but is missing from `contracts/components.contract.json`.
- `ComponentHierarchy` is exported and used in installed-base guidance, but is missing from `contracts/components.contract.json`.
- `ComponentHierarchyItem` is exported but is missing from `contracts/components.contract.json`.
- `MetricStrip` is exported from `components`, but the contract classifies it under `decision`.

### P0 — Story mapping is not machine-audited

Storybook can load stories from `src/**/*.stories.*`, but the repo does not yet expose a simple component-to-story map.

For v0.5.1, create a verifiable story coverage map or test.

Required output:

```txt
exported component → story file or explicit no-story rationale
```

### P0 — Documentation format is not GenAI-standardized

Most guideline files predate the v0.5.1 standard.

Every component guideline should eventually use this structure:

```txt
Purpose
Use when
Do not use when
Use instead
Required props
Controlled values
GenAI generation rules
Accessibility rules
Evidence and trust rules
Common anti-patterns
Good example
Bad example
Related stories
Related contracts
```
