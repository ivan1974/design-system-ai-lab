# v0.6.0 Phase 12 — Legacy and simplification audit

## Purpose

This audit starts Phase 12 before v0.7.0.

The goal is to identify remaining legacy, duplicated guidance, obsolete examples and deprecated values that could keep the project in a mixed v0.5.x / v0.6.0 state.

Because the project is not in production, unused legacy should be deleted rather than preserved by default.

## Decision rule

```txt
If a file is not needed for v0.6.0 runtime, reference, evaluation, contracts, tests, benchmarks or source rationale, delete it.
```

Do not keep files only because they may be useful later.

If a file is useful for v0.7.0, it must be explicitly marked as v0.7.0 planning or moved into a source/backlog location.

---

## Classification model

| Classification | Meaning | Default action |
| --- | --- | --- |
| runtime-facing | Read by Figma Make during normal generation | Keep short |
| reference-facing | Read only when selecting or checking a component/pattern | Keep if aligned |
| evaluation-facing | Used for review or repair | Keep if routed/tested |
| contract-facing | Machine-readable rules or package contracts | Keep and align version |
| source-non-runtime | Rationale, domain knowledge or principles | Keep only if clearly non-runtime |
| test-only | Required by tests or fixtures | Keep if tests are current |
| archive | Historical, not active | Keep only if intentionally historical |
| delete | Obsolete and not needed | Remove |

## Phase 12A conclusion

The repo is structurally close to v0.6.0, but still carries a v0.5.x editorial layer.

The strongest cleanup opportunities are:

```txt
README rewrite
prompt folder reduction
golden examples reduction
deprecated alias removal
legacy review docs deletion
migration/release docs deletion
make-minimal-contract deletion
source non-runtime labeling
```

This should happen before v0.7.0 starts, otherwise v0.7.0 will inherit two competing narratives.
