## Semantic ownership rules

### Action components

```txt
ActionCard → one emphasized owned action
ActionRow → dense repeated owned action
ActionList → small grouped action set
CreateActionDialog → standard action creation flow
```

All owned action displays must preserve:

```txt
owner
due date
priority
```

`PriorityPill` alone is not enough to represent follow-through.

---

### Status and metadata components

```txt
StatusPill → short passive status
SemanticTag → lightweight semantic metadata
Badge → lightweight metadata from basic components
PriorityPill → action priority only
SourceStrengthPill → source basis only
```

These components are not interchangeable.

---

### Signal, evidence and recommendation components

```txt
SignalRow → observed signal and value
EvidenceRow → source-backed evidence with trust context
RecommendationCard → interpreted recommendation with rationale and readiness
AlertCard → attention item with severity and recommendation
ValueProofCard → proof maturity and proof points
```

Observed signal, evidence, recommendation and proof each have distinct UI ownership.

---

### Metrics and priority components

```txt
MetricGrid → small set of decision-relevant metrics
PriorityList → risks, blockers or attention items to review first
StatusSummary → compact display-only status context
```

`MetricGrid` must remain tied to a decision.

`PriorityList` must remain attention-oriented.

`StatusSummary` must remain display-only context.

## Controlled values reinforced

Guidelines now reference controlled values from:

```txt
contracts/props.contract.json
contracts/evidence-and-trust.contract.json
```

Examples:

```txt
priority: low | medium | high | critical
status: todo | in-progress | blocked | done | in_progress
status tone: neutral | primary | success | warning | danger
sourceStrength: unknown | partial | single-source | multi-source | validated
proofReadiness: not-available | expected-only | internal-proof | customer-ready-proof
validationStatus: not-reviewed | internal-review-needed | internally-validated | customer-ready | blocked
```

Deprecated aliases remain documented only where source types accept them.

---

## Anti-overlap rules added

```txt
PriorityPill does not replace ActionRow or ActionCard.
StatusPill does not replace SourceStrengthPill.
SemanticTag does not replace StatusPill when a formal status is needed.
SourceStrengthPill represents source basis only.
SignalRow remains observation-only.
MetricGrid remains decision-metric focused.
PriorityList remains attention-priority focused.
StatusSummary remains compact display context.
```
