# v0.6.0 Reference guidelines template audit

## Purpose

This mini-audit checks whether `guidelines/reference/` is aligned with `guidelines/templates/component-guideline-template.md` and with the current generation-rule tests.

It is intentionally diagnostic. It does not require every reference file to be rewritten before v0.7.0.

## Template standard

The v0.6.0 component guideline template says that every GenAI-ready guideline should contain:

```txt
Purpose
Use this component when
Do not use this component when
Prefer this component over
Never generate
Required props
Controlled values
GenAI generation rules
Common generation failures
Repair prompt
Related stories
Related contracts
```

Optional sections may be added when relevant:

```txt
Evidence and trust rules
Actionability rules
Accessibility rules
Business intent
Domain model alignment
```

## Current test coverage

The active test is:

```txt
tests/generation-rules/documentation-contract-alignment.test.ts
```

It enforces the full template only for registry entries with:

```txt
genAIStatus: preferred
```

It also checks:

- preferred components have a guideline path;
- controlled values appear in the guideline when documented in `props.contract.json`;
- reference guidelines do not recommend unknown PascalCase package components;
- active prompts do not reference obsolete components.

This means the repo is test-aligned for preferred runtime components, but not fully template-aligned across all reference files.

## Recommended interpretation

Do not treat every reference file as equal.

Use three levels:

### Level 1 — Runtime preferred

Files used by `preferred` registry entries.

Rule:

```txt
Must follow the full template and must pass tests.
```

Status:

```txt
Aligned after queue-rows.md fix.
```

### Level 2 — Runtime allowed / use-with-care

Files used by `allowed` or `use-with-care` entries.

Rule:

```txt
Should follow the template when the component is likely to be generated by Make.
Can be shorter if the component is secondary or only used inside patterns.
```

Status:

```txt
Partially aligned. Good candidates for gradual cleanup before v0.7.0.
```

### Level 3 — Internal, deprecated, blocked or background

Files used by `internal-only`, `deprecated`, `blocked-for-genai`, or broad architecture explanation.

Rule:

```txt
Should not necessarily use the full template.
Should clearly state status, replacement and boundaries.
```

Status:

```txt
Keep, simplify or mark clearly. Do not force the full template unless the file becomes GenAI-facing.
```

## Test alignment recommendation

Current tests are correctly scoped for v0.6.0.

Do not immediately extend the full-template test to every file in `guidelines/reference/` because that would force explanatory and deprecated files into an artificial component format.

Recommended next test improvement:

```txt
1. Keep full template enforcement for preferred entries.
2. Add lighter status-boundary checks for allowed, use-with-care, internal-only, deprecated and blocked-for-genai entries.
3. Add a test that any non-preferred guideline used in runtime must explicitly say its status and generation boundary.
```

## Mini-audit conclusion

`guidelines/reference/` is aligned enough for the current v0.6.0 tests.

The preferred runtime layer is now template-aligned.

The full reference folder is not uniformly template-aligned, and that is acceptable if we keep the distinction between:

```txt
runtime preferred guidance
allowed/use-with-care guidance
internal/deprecated/background reference
```

Before v0.7.0, the best improvement would not be to rewrite every file. It would be to add a lighter status/boundary standard for non-preferred reference files and gradually align high-value allowed/use-with-care guidelines.
