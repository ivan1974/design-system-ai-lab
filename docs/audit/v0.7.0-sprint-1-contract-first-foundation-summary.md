## Contract decisions

### Closed screen contract

Installed Base Intelligence is treated as a closed screen contract.

Undefined elements are not allowed.

The screen must preserve exact layers, zones, filters, columns, tabs, actions, asset states, interaction rules and forbidden content.

### Page Header inconsistency

The reference specification says the Page Header contains exactly four elements but lists five.

Sprint 1 resolves this as:

```txt
Page Header contains exactly five elements.
```

The five required elements are:

1. Context Pill
2. Site Context
3. Portfolio Title
4. Portfolio Scope Summary
5. Asset Intelligence

### Progressive decision disclosure

Sprint 1 embeds the v0.7.0 principle:

```txt
Generated decision screens must reveal decision-critical information first, then expose evidence and explanation progressively.
```

Required levels:

1. Signal
2. Decision
3. Evidence

### Third-party asset rule

Third-party assets must always display:

| Field | Value |
| --- | --- |
| Connectivity | Unknown |
| Coverage | No Coverage |
| Health | Unknown |
| Status | No record |

No exception is allowed.
