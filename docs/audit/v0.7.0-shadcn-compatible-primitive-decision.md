# v0.7.0 shadcn-compatible primitive decision

## Status

Accepted for v0.7.0 planning.

## Decision

The Design System may rebuild or adapt its primitive implementation layer using shadcn-compatible Radix and Tailwind patterns.

The public generation surface remains `design-system-ai-lab`.

Figma Make must never import local `components/ui` files, internal primitive files, shadcn files or Radix primitives directly in generated examples.

## Rationale

Figma Make generated projects can include shadcn-like files such as `default_shadcn_theme.css`, local `components/ui` primitives and Radix dependencies.

Aligning internal DS primitives with this implementation model should make future component work faster and more natural for AI generation.

However, the value of this package is not a primitive library. The value is the contract system that tells Figma Make what decisions, states, layers, props and screen structures are allowed.

## Architecture rule

```txt
shadcn-compatible implementation internally
contracted DS surface externally
```

## Non-negotiable rules

- Generated examples import only from `design-system-ai-lab`.
- Generated examples import styles only from `design-system-ai-lab/styles.css`.
- Golden examples must not import `components/ui`.
- Golden examples must not import `@radix-ui/*` directly.
- Business patterns must wrap primitives instead of exposing primitive internals.
- Props contracts remain the public API source of truth.
- Screen contracts override primitive defaults.
- DS tokens remain the visual source of truth.

## Migration posture

Because the package is not in production, the primitive layer can be refactored radically.

The migration should prioritize primitives that unlock future screen work:

- Button
- Badge
- Dialog
- Sheet or SidePanel
- Tabs
- Select
- Input
- Checkbox
- Table
- ScrollArea
- Tooltip
- Popover
- Command or search

## What must not change

The public Make-facing rule does not change:

```txt
Use public design-system-ai-lab exports only.
Do not create local components.
Do not import internal primitives.
Do not bypass contracts.
```
