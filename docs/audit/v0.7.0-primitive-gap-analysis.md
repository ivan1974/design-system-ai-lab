# v0.7.0 primitive gap analysis

## Status

Sprint 3.5 planning artifact.

## Purpose

Identify which existing DS primitives should be kept, adapted or rebuilt using shadcn-compatible Radix and Tailwind implementation patterns.

This audit does not authorize generated screens to import shadcn, Radix or local `components/ui` files.

## Decision matrix

| DS primitive | Current status | shadcn-like equivalent | Decision | Reason |
| --- | --- | --- | --- | --- |
| Button | exists | Button | adapt | Core primitive; should align with familiar variant and size pattern while preserving DS API. |
| Badge | exists | Badge | adapt | Useful base for tags and labels; must remain token-first. |
| Dialog | exists | Dialog | adapt | Radix dialog behavior is already compatible with accessible overlays. |
| SlideOverPanel | exists | Sheet | replace or reduce | Lower-level primitive can become implementation detail behind SidePanel. |
| SidePanel | exists | Sheet | keep public, rebuild underneath | Public contract is useful; implementation can align with shadcn Sheet behavior. |
| Tabs | exists | Tabs | adapt | Needed for detail panels and screen contracts. |
| Select | exists | Select | adapt | Needed for filters and controlled values. |
| Input | exists | Input | adapt | Needed for search and forms. |
| Label | exists | Label | adapt | Keep with Field and forms. |
| Field | exists | Field | adapt | Align with form composition patterns. |
| Textarea | exists | Textarea | adapt | Lower priority but cheap to align. |
| Divider | exists | Separator | adapt or rename later | Keep public Divider; implementation may align with Separator. |
| Card | exists | Card | use with care | Keep but avoid repeated-object card stacks. |
| Surface | exists | Card/Surface | keep | DS-specific surface abstraction remains useful. |
| Table | missing | Table | add | Needed for installed base list and operational inventory. |
| ScrollArea | missing | Scroll Area | add | Needed for panels and dense workspaces. |
| Checkbox | missing | Checkbox | add | Needed for advanced filters. |
| Tooltip | missing | Tooltip | add later | Useful, not immediately required by Installed Base contract. |
| Popover | missing | Popover | add later | Useful for filters/search, not first blocker. |
| Command | missing | Command | add later | Useful for search/command palette, not first blocker. |

## Priority for Sprint 3.5 migration

1. Document the primitive architecture and tests.
2. Add missing foundational primitives needed by upcoming sprints: Table, ScrollArea, Checkbox.
3. Rebuild or adapt SidePanel on top of shadcn-compatible Sheet behavior.
4. Audit Button, Badge, Dialog, Tabs, Select and Input before replacing implementation.
5. Keep business primitives and patterns as wrappers over the primitive layer.

## Non-goals

- Do not migrate business components directly to shadcn APIs.
- Do not expose `components/ui` in the package surface.
- Do not allow Make to import Radix directly.
- Do not dilute screen contracts into primitive decisions.

## Migration guardrails

Every migrated primitive must keep or update:

- public export;
- props contract;
- component registry entry;
- guideline;
- story coverage;
- no local design-system checks;
- no internal import checks;
- package import contract.
