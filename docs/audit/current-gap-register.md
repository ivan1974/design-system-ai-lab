# Current Gap Register — v0.6.0

## Purpose

This register tracks the gaps that must be closed for `design-system-ai-lab` v0.6.0.

v0.6.0 focuses on GenAI runtime optimization and production readiness.

It does not expand the component library.

---

## Severity scale

| Severity | Meaning |
|---|---|
| P0 | Blocks v0.6.0 release |
| P1 | Required before v0.6.0 release candidate |
| P2 | Should be completed if low risk |
| P3 | Defer to v0.7.0 or later |

## Deferred to v0.7.0

The following work is explicitly out of scope for v0.6.0:

```txt
full component-level progressive disclosure
summary / standard / expanded modes
large refactor of decision components
large refactor of business patterns
human comprehension testing
new business domains
new visual brand expansion
```

---

## Acceptance rule

A gap can be closed only when one of the following is true:

- the relevant file has been created or updated
- the relevant contract exists
- the relevant test exists
- the relevant legacy path has been archived or removed from runtime
- the gap has been explicitly deferred with rationale
