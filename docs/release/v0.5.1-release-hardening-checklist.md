# v0.5.1 Release Hardening Checklist

## Purpose

Use this checklist before publishing `design-system-ai-lab@0.5.1`.

Version `0.5.1` is a patch release focused on GenAI generation quality, documentation alignment and package guidance assets.

---

## Release principle

```txt
No component, pattern, prompt, example or review rule should teach an AI to generate something the contracts would reject.
```

---

## Pre-release validation

Run locally:

```bash
npm run check
npm run build-storybook
npm pack --dry-run
```

Recommended expanded validation:

```bash
npm run lint:md
npm run test
npm run build:ds
npm run build-storybook
npm pack --dry-run
```

---

## Publish flow

Publish first with the `next` tag:

```bash
npm publish --tag next
```

Validate in a consuming app:

```bash
npm install design-system-ai-lab@next
```

Validate with Figma Make using:

```txt
guidelines/make-minimal-contract.md
guidelines/Guidelines.md
guidelines/prompts/*
guidelines/examples/golden/*
benchmarks/figma-make/cases/*
```

Promote only after validation:

```bash
npm dist-tag add design-system-ai-lab@0.5.1 latest
```

---

## Acceptance criteria

The release can be promoted when:

```txt
npm run check passes
npm run build-storybook passes
npm pack --dry-run contains hardened guidance assets
consumer app imports package successfully
Figma Make generation follows v0.5.1 contracts and prompts
review and repair loop routes failures to actual repair prompts
```
