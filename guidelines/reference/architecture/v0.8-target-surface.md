# v0.8 Target Surface Architecture

## Purpose

This guideline defines the v0.8 Make-facing target surface.

It is the architectural source for the regeneration phase. The goal is not to preserve every existing component. The goal is to define the smaller surface that the design system must converge to.

## Strategic rule

```txt
Target surface first.
Guidelines and contracts second.
Code regeneration third.
Legacy deletion last.
```

## Source contract

Read first:

```txt
contracts/v0.8-target-surface.contract.json
```

Then align:

```txt
contracts/components.contract.json
contracts/component-registry.contract.json
contracts/props.contract.json
contracts/tokens.contract.json
contracts/visual-rules.contract.json
```

## Core principle

No component can remain in the Make Kit surface only because it already exists.

A component must justify at least one of these:

```txt
distinct visual structure
distinct interaction responsibility
reusable composition role
strict screen-contract responsibility
```

Same visual structure means same component.

Different business content must be handled by content, props, mappings, slots and contracts.

## Target foundations

Use:

```txt
src/design-system/foundations/tokens.css
src/design-system/styles.css
guidelines/tokens.md
guidelines/styles.md
guidelines/runtime/visual-rules.md
```

Do not create:

```txt
src/design-system/tokens.css
```

## Target semantic display surface

Use only:

```txt
SemanticTag
SemanticPill
StatusIndicator
MetaLabel
```

Do not recommend these as public Make-facing components:

```txt
HealthPill
CoverageTag
DppTag
ConnectivityLabel
StatusWithIcon
PriorityPill
PriorityLabel
ProofStatus
SourceStrengthPill
StatusPill
```

Temporary legacy wrappers may remain in source code only until regeneration, but they must not be preferred, documented as public primitives, used by golden examples or used as positive benchmark examples.

## Target card and decision surface

Use this target grammar:

```txt
Card
MetricBlock
DecisionBlock
EvidenceBlock
ActionBlock
StatusSummary
ReviewQueueRow
ActionRow
EvidenceRow
```

Do not keep one public component per business label when the structure is the same.

Likely merges:

```txt
MetricCard -> MetricBlock
AlertCard -> DecisionBlock
RecommendationCard -> DecisionBlock
ActionCard -> ActionBlock
EvidenceCard -> EvidenceBlock
InsightCard -> DecisionBlock or Card
RiskCard -> DecisionBlock or StatusSummary
```

## Target composition surface

Use:

```txt
WorkspaceShell
MainNavigation
PageHeading
FilterBar
MasterDetailLayout
WorkspaceDetailPanel
SidePanel
SectionStack
SectionBlock
ListContainer
StickyActionBar
```

A composition component remains public only if it helps Make build a reusable screen structure.

## Target screen patterns

The target public screen patterns are:

```txt
Installed Base Intelligence
Customer Monitoring
```

Installed Base remains the strict contractual screen.

Customer Monitoring becomes the second generation proof.

Renewal Risk Review and Asset Recommendation Review remain active examples until the target surface proves it can support them without separate visual grammars.

## Runtime generation rule

Generated screens must use the target surface, not legacy wrappers.

Generated screens must not introduce local components to compensate for deleted components.

Generated screens must not recreate deleted components under new names.

## Regeneration rule

After the target contracts and guidelines are stable, regenerate or rewrite:

```txt
src/design-system/components/
src/design-system/composition/
src/design-system/decision/
src/design-system/forms/
src/design-system/patterns/
src/design-system/stories/
src/design-system/index.ts
```

Do not spend time polishing legacy components that will be removed unless they block transition tests.

## Exit criteria

The reset is complete when:

```txt
the target surface contract exists
the target architecture guideline exists
runtime rules point to the target surface
obsolete semantic wrappers are blocked from Make-facing guidance
obsolete card wrappers are marked for merge or deletion
golden examples are ready to be regenerated from the target surface
```
