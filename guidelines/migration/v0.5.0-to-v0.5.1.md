# Migration Notes — v0.5.0 to v0.5.1

## Purpose

Version `0.5.1` is a patch release for GenAI generation quality.

It hardens documentation, contracts, prompts, examples, review checklists and repair routing.

It does not introduce a breaking runtime component API.

---

## Who should read this

Read this note if you use the package for:

```txt
Figma Make generation
AI-assisted interface generation
golden example validation
design-system documentation review
benchmark scoring
```

---

## Runtime migration

No runtime migration is expected for consuming React apps.

Continue importing components from:

```tsx
import { Button, WorkspaceShell } from "design-system-ai-lab";
import "design-system-ai-lab/styles.css";
```

Do not import from internal package paths.

---

## Generation guidance migration

For new Figma Make or GenAI generations, use this order:

```txt
1. guidelines/make-minimal-contract.md
2. guidelines/Guidelines.md
3. guidelines/setup.md
4. guidelines/prompts/*
5. contracts/components.contract.json
6. contracts/props.contract.json
7. contracts/evidence-and-trust.contract.json
8. guidelines/evaluation/review/blocking-checklist.md
9. guidelines/evaluation/repair/repair-router.md
```

---

## What changed

Version `0.5.1` adds or hardens:

```txt
priority component guidelines
controlled values and domain models
golden examples
reusable prompts
benchmark prompt alignment
review checklists
repair prompts and repair router
documentation drift tests
package content coverage
```

---

## What to update in prompts

Avoid prompts that ask Make to generate:

```txt
local components
local wrappers
card stacks
raw form controls
generic dashboards
AI confidence as source strength
expected outcomes as proven value
customer-ready proof without validation
```

Prefer prompts that ask for:

```txt
WorkspaceShell
PageHeading
FilterBar
MasterDetailLayout
ListContainer
WorkspaceDetailPanel
queue rows
EvidenceRow
ActionRow
SourceStrengthPill
ValueProofCard
repair-router.md when review fails
```

---

## Review migration

Replace older ad hoc review with the v0.5.1 review loop:

```txt
blocking-checklist.md
→ repair-router.md when needed
→ workspace-v2-checklist.md
→ quality-checklist.md
→ closest golden example comparison
```

---

## Acceptance principle

A v0.5.1 generated screen should not teach an AI to generate something that contracts or tests would reject.

Use this rule when reviewing generated output:

```txt
If a prompt, example, guideline or review rule conflicts with contracts, the contract wins.
```
