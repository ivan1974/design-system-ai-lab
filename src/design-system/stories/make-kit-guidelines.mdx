# Make Kit Guidelines — v0.3.0

This Storybook page is a short visual entry point for the Make kit.

The canonical Make-ready documentation lives in:

```txt
guidelines/
```

Use Storybook to review examples visually.

Use `guidelines/` to instruct Figma Make.

---

## Core principle

The design system is not a card stack generator.

Figma Make should generate decision workspaces, not generic dashboards.

Preferred structure:

```txt
WorkspaceShell
→ FilterBar
→ MasterDetailLayout when list/detail review is needed
→ DetailPanel / DetailPanelTabs / StickyActionBar for selected-item review
→ SectionStack / SectionBlock for dense grouping
→ rows and compact primitives for repeated information
```

---

## Required package usage

Generated screens must import from the package root:

```tsx
import {
  WorkspaceShell,
  FilterBar,
  MasterDetailLayout,
  DetailPanel,
  ActionRow,
} from "design-system-ai-lab";
```

Generated screens must import styles once:

```tsx
import "design-system-ai-lab/styles.css";
```

Do not import from internal package paths.

Do not create local design system folders.

Do not recreate package components locally.

## Demo claim

Do not claim that Figma Make replaces designers.

Correct claim:

> AI becomes more useful when the design system provides the vocabulary, rules,
> structure and repair prompts for generation.

---

## Final principle

Figma Make should not generate from imagination.

It should compose inside the system.
