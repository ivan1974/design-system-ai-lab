# Figma Make Setup — v0.3.0

This Storybook page is a short setup checklist.

The canonical setup documentation lives in:

```txt
guidelines/setup.md
```

---

## Package setup

A consuming Make / React project should use the published package:

```txt
design-system-ai-lab
```

Generated screens should import components from the package root:

```tsx
import {
  WorkspaceShell,
  FilterBar,
  MasterDetailLayout,
  DetailPanel,
  ActionRow,
} from "design-system-ai-lab";
```

Generated screens should import styles once:

```tsx
import "design-system-ai-lab/styles.css";
```

---

## Do not create local copies

Figma Make must not create:

```txt
packages/design-system-ai-lab/
src/components/ui/
local Button wrappers
local Card wrappers
local Badge wrappers
local design-system copies
```

Do not import from:

```txt
design-system-ai-lab/dist/*
design-system-ai-lab/src/*
./components/ui/*
```

## Workspace setup expectation

The expected v0.3.0 structure is:

```txt
WorkspaceShell
→ FilterBar
→ MasterDetailLayout when list/detail review is needed
→ DetailPanel / DetailPanelTabs / StickyActionBar for selected item review
→ rows and compact primitives for repeated information
```

Do not generate a generic dashboard or a long stack of cards.

## Final principle

Figma Make should behave as a screen composition assistant inside a governed design system.

It should not behave as an unrestricted visual designer.
