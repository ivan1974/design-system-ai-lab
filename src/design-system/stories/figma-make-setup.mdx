# Figma Make Setup — v0.4.0

This Storybook page is a short setup checklist.

The canonical setup documentation lives in:

```txt
guidelines/setup.md
```

---

## Package setup

A consuming Make / React project should use the published package:

```txt
design-system-ai-lab
```

Generated screens should import components from the package root:

```tsx
import {
  WorkspaceShell,
  PageHeading,
  SectionHeading,
  FilterBar,
  MasterDetailLayout,
  WorkspaceDetailPanel,
  ListContainer,
  AssetQueueRow,
  Tabs,
  StatusPill,
  ActionRow,
} from "design-system-ai-lab";
```

Generated screens should import styles once:

```tsx
import "design-system-ai-lab/styles.css";
```

Do not import from internal package paths.

---

## Do not create local copies

Figma Make must not create:

```txt
packages/design-system-ai-lab/
src/components/ui/
local Button wrappers
local Card wrappers
local Badge wrappers
local Tabs wrappers
local Drawer or SidePanel wrappers
local QueueRow wrappers
local design-system copies
```

Do not import from:

```txt
design-system-ai-lab/dist/*
design-system-ai-lab/src/*
./components/ui/*
```

## v0.4.0 workspace setup expectation

The expected v0.4.0 structure is:

```txt
WorkspaceShell
→ PageHeading
→ FilterBar, SecondaryNavigation or SegmentedControl for scope
→ MasterDetailLayout when list/detail review is needed
→ ListContainer with QueueRows for repeated review objects
→ WorkspaceDetailPanel for selected item detail
→ Tabs inside detail when needed
→ StickyActionBar or ActionRow for follow-through
```

Use business patterns when the section intent matches:

```txt
CustomerStatusCard
CustomerReviewReadinessCard
ConnectivityCoverageCard
AssetIntelligenceSummary
RenewalRiskSummary
ValueProofCard
ServiceRiskSummary
RecommendationReviewPanel
```

Do not generate a generic dashboard or a long stack of cards.

## Final principle

Figma Make should behave as a screen composition assistant inside a governed design system.

It should not behave as an unrestricted visual designer.
