# AI Usage Guide — v0.3.0

This Storybook page is a short orientation guide.

The canonical AI and Make rules live in:

```txt
guidelines/Guidelines.md
guidelines/principles/ai-usage.md
guidelines/principles/evidence-and-trust.md

```

---

## Core message

The AI should not generate from imagination.

It should compose first drafts from:

- the package vocabulary
- the design tokens
- the screen architecture rules
- the business patterns
- the prompt constraints
- the evidence and trust rules
- the golden examples

---

## BI-first, AI-assisted

Use this model:

```txt
BI shows what is happening.
GenAI helps explain what it means and what to do next.
Workflow helps users execute.
Human validation protects critical decisions.
```

Do not use AI for basic retrieval:

```txt
customer name
contract date
owner
status
counts
telemetry
standard KPIs
source data
```

Use AI only for:

```txt
synthesis
explanation
prioritization
recommendation
drafting
reformulation
proof gap explanation
customer-ready wording after evidence is visible
```

---

## Trust rules

Generated screens should show:

```txt
facts
→ source scope
→ source strength
→ signals
→ recommendations
→ actions
→ validation status
```

Do not claim:

- AI validation
- automatic approval
- proven value without evidence
- customer-ready proof without validation
- complete asset visibility when visibility is partial

---

## Workspace v2 link

For v0.3.0, AI-generated screens should follow the decision workspace standard:

```txt
WorkspaceShell
→ FilterBar
→ MasterDetailLayout
→ DetailPanel
→ evidence and actions
```

See:

```txt
guidelines/prompts/workspace-v2.md
```

---

## What to inspect in Storybook

Use:

```txt
Design System / Golden Examples
```

to judge whether generated screens preserve:

- evidence hierarchy
- source visibility
- human validation
- actionable follow-up
- workspace structure
- low card saturation

---

## Final principle

AI can accelerate first draft production.

It does not replace design judgment.
