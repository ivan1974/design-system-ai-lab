This Storybook page is a short orientation guide.

---

## Core message

AI should compose inside the design system.

It should not generate from imagination or create a competing visual system.

A useful AI-generated screen should combine:

- the package vocabulary
- the v0.4.0 workspace structure
- the white-first visual system
- the approved decision components
- the approved business patterns
- the evidence and trust rules
- the repair prompts
- the golden examples

---

## BI-first, AI-assisted

Use this model:

```txt
BI shows what is happening.
GenAI helps explain what it may mean and what to do next.
Workflow helps users execute.
Human validation protects critical decisions.
```

Do not use AI for basic retrieval:

```txt
customer name
contract date
owner
status
counts
telemetry
standard KPIs
source data
```

Use AI only for:

```txt
synthesis
explanation
prioritization
recommendation drafting
reformulation
proof gap explanation
customer-ready wording after evidence is visible
```

---

## v0.4.0 workspace expectation

AI-generated screens should follow the v0.4.0 decision workspace standard:

```txt
PageHeading
→ FilterBar or SecondaryNavigation
→ MasterDetailLayout
→ ListContainer with QueueRows
→ WorkspaceDetailPanel
→ Tabs when detail has multiple review questions
→ EvidenceRow, RecommendationReviewPanel, RecommendationCard and ActionRow when relevant
```

Use business patterns before generic cards when the section intent matches:

```txt
CustomerStatusCard
CustomerReviewReadinessCard
ConnectivityCoverageCard
AssetIntelligenceSummary
RenewalRiskSummary
ValueProofCard
ServiceRiskSummary
RecommendationReviewPanel
```

---

## Trust rules

Generated screens should make this sequence visible:

```txt
facts
→ source scope
→ source strength
→ signals
→ interpretation or recommendation
→ validation status
→ action
```

Do not claim:

- AI validation
- automatic approval
- proven value without evidence
- customer-ready proof without validation
- complete asset visibility when visibility is partial
- non-connected assets as live-monitored assets

---

## What to inspect in Storybook

Use:

```txt
Design System / Golden Examples
```

to judge whether generated screens preserve:

- PageHeading and SectionHeading hierarchy
- white-first visual style
- row density with ListContainer and QueueRows
- WorkspaceDetailPanel for selected detail
- source visibility
- human validation
- actionable follow-up
- low card saturation

## Final principle

AI can accelerate first draft production.

It does not replace design judgment, source validation or the governed design system vocabulary.
